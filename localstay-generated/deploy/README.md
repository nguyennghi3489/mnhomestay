# 🏠 LocalStay MVP - Deployment Guide

This deployment package contains everything needed to run LocalStay MVP in development or production environments using Docker.

## 🚀 Quick Start

### Development Environment

```bash
cd deploy
./scripts/setup-dev.sh
```

Your LocalStay application will be available at:
- **Frontend**: http://localhost
- **Backend API**: http://localhost:3000  
- **Database**: localhost:5432 (user: localstay, db: localstay)
- **Redis**: localhost:6379

### Production Deployment

```bash
cd deploy
cp env/.env.production .env
# Edit .env with your production settings
./scripts/deploy-prod.sh
```

## ⚙️ Configuration

### Development
Development uses `env/.env.development` with safe defaults for local development.

### Production Setup
1. Copy the production template: `cp env/.env.production .env`
2. Edit `.env` and change ALL placeholder values:
   - `DATABASE_URL`: Your production database connection
   - `JWT_SECRET`: Generate a secure random string
   - `FRONTEND_URL` / `API_URL`: Your actual domain names
3. Run deployment: `./scripts/deploy-prod.sh`

## 🔒 Security Checklist

Before deploying to production:

- [ ] Change `JWT_SECRET` to a secure random string (32+ characters)
- [ ] Update database credentials (`DATABASE_URL`)
- [ ] Configure your actual domain names
- [ ] Set up SSL certificates (consider using Cloudflare or Let's Encrypt)
- [ ] Review and update CORS settings
- [ ] Disable debug features (`ENABLE_SWAGGER=false`)

## 🚨 Troubleshooting

### Services won't start
```bash
./scripts/status.sh                    # Check current status
docker-compose logs -f backend         # Check backend logs
docker-compose logs -f frontend        # Check frontend logs  
docker-compose logs -f db              # Check database logs
```

### Database connection issues
```bash
docker-compose exec db pg_isready -U localstay    # Test database health
docker-compose exec backend npm run db:migrate   # Run migrations
```

### Port conflicts
If you get port conflict errors, edit `docker-compose.yml` and change the port mappings:
```yaml
ports:
  - "8080:80"    # Frontend (change 80 to 8080)
  - "3001:3000"  # Backend (change 3000 to 3001)
```
